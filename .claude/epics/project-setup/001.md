# Task: Initialize Next.js Application with TypeScript and Tailwind

## Description

Create a new Next.js 15 project from scratch using the App Router architecture with TypeScript and Tailwind CSS configured. This establishes the foundation for the entire Korella CRM application with mobile-first design patterns and all core development dependencies.

## Acceptance Criteria

- [ ] Next.js 15 project created with App Router (not Pages Router)
- [ ] TypeScript configured with strict mode enabled
- [ ] Tailwind CSS v4 installed and operational
- [ ] Mobile-first breakpoints defined (xs: 375px, sm: 640px, md: 768px, lg: 1024px)
- [ ] ESLint and Prettier configured for code quality
- [ ] Core dependencies installed (React Hook Form, Zod, TanStack Query, Zustand)
- [ ] Development server starts successfully with `npm run dev`
- [ ] Application loads at http://localhost:3000
- [ ] No TypeScript compilation errors
- [ ] Hot reload works (edit file, see changes immediately)

## Technical Details

### Implementation Steps

**1. Initialize Next.js Project**

```bash
# Create Next.js 15 project with TypeScript
npx create-next-app@latest .

# Interactive prompts - select these options:
# ✅ TypeScript: Yes
# ✅ ESLint: Yes
# ✅ Tailwind CSS: Yes
# ✅ `src/` directory: No (use app/ directly)
# ✅ App Router: Yes
# ✅ Customize default import alias: No (@/* is fine)
```

**2. Install Core Dependencies**

```bash

# State management and data fetching
npm install @tanstack/react-query zustand

# Form handling and validation
npm install react-hook-form zod @hookform/resolvers

# Supabase client (for later tasks)
npm install @supabase/supabase-js

# Date utilities
npm install date-fns

# PWA support
npm install next-pwa
```

**3. Install Development Dependencies**

```bash
npm install -D prettier prettier-plugin-tailwindcss @types/node
```

**4. Configure Tailwind with Mobile-First Breakpoints**

Update `tailwind.config.ts`:

```typescript
import type { Config } from 'tailwindcss';

const config: Config = {
  content: [
    './pages/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}',
    './app/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  theme: {
    screens: {
      xs: '375px', // iPhone SE (minimum requirement)
      sm: '640px', // Large phones
      md: '768px', // Tablets
      lg: '1024px', // Desktop
      xl: '1280px', // Large desktop
    },
    extend: {
      minHeight: {
        touch: '44px', // Minimum touch target (Apple HIG)
      },
      minWidth: {
        touch: '44px',
      },
    },
  },
  plugins: [],
};
export default config;
```

**5. Configure TypeScript Strict Mode**

Update `tsconfig.json`:

```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true
    // ... other settings from create-next-app
  }
}
```

**6. Configure Prettier**

Create `.prettierrc`:

```json
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 100,
  "tabWidth": 2,
  "plugins": ["prettier-plugin-tailwindcss"]
}
```

**7. Update `.gitignore`**

Add to `.gitignore`:

```
# Environment variables
.env.local
.env.*.local

# Next.js
.next/
out/

# Dependencies
node_modules/

# IDE
.vscode/
.idea/

# OS
.DS_Store
```

**8. Test Development Server**

```bash
npm run dev
```

Visit http://localhost:3000 - should see Next.js welcome page.

### Configuration Notes

**Next.js 15 Features**:

- App Router (not Pages Router)
- Server Components by default
- Client Components use `'use client'` directive
- Built-in API routes: `app/api/`

**TypeScript Strict Mode Benefits**:

- Catches more errors at compile time
- Better IDE autocomplete
- Safer refactoring

**Tailwind Mobile-First**:

- Default styles for xs (375px)
- Use `sm:`, `md:`, `lg:` for larger screens
- Touch targets minimum 44x44px

## Troubleshooting

**Issue**: `npm run dev` fails with port 3000 already in use

**Solution**:

```bash
# Kill process on port 3000
npx kill-port 3000
# Or use different port
npm run dev -- -p 3001
```

**Issue**: TypeScript errors in node_modules

**Solution**:

```bash
# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

**Issue**: Tailwind styles not applying

**Solution**:

- Verify `tailwind.config.ts` content paths include your files
- Restart dev server
- Check browser console for CSS loading errors
